Project description

We have a project which is about developing a proof of concept (POC) of a web based .epub reader. It is mandatory for the reader to render the ePub files on all kind of devices, OS and browsers: - Devices - mobile, tablet, desktop - OS - Windows, Mac, Chromebook, Ubuntu - Browsers - Chrome, Safari, Firefox, IE 9+, Opera

Here is a Brief info on the project:
https://docs.google.com/document/d/1CFY0lSXSpDLBLeYgmy67s6OlP9t4ZEgN-tcohYJGe3c/edit?usp=sharing

Please suggest on what technologies do you suggest to use for the development. We can suggest these, but you are welcome to suggest others: - ePub.js - Monocle (https://github.com/joseph/Monocle) - Custom solution where the epub format is “converted” - for example the HTML/CSS content is extracted from the files and displayed as part of the reader’s HTML page

Here is what has been done by now by our internal developer with ePub.js (It is not mandatory to proceed with ePub.js library): epub.rainbowriders.dk

The main issues that we have by now are: - It takes a lot of time to generate the total number of pages , which makes the overall performance of the viewer too slow - The solution still doesn’t work on IE9+ - The library doesn’t seem flexible enough for the features in the POC phase 2

Skills required

CSS, HTML5, Javascript
